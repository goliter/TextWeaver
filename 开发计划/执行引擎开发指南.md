# 执行引擎开发指南

## 项目现状

- 前端已实现工作流画布，支持创建节点和连线
- 后端已实现工作流、节点、边的 CRUD 操作
- 后端已实现执行记录和执行日志的 CRUD 操作
- 数据模型已定义完成，包括 Flow、Node、Edge、Execution 和 ExecutionLog
- 执行引擎尚未实现，目前只是创建执行记录，没有实际执行工作流

## 开发目标

构建一个简单的 MVP 执行引擎，支持以下功能：

- 从开始节点开始执行工作流
- 执行 AI 节点，使用预设的 AI 模型和提示词
- 将 AI 返回结果显示在执行日志中
- 执行结束节点，完成工作流

## 开发步骤

### 1. 实现执行引擎核心逻辑

1. **创建执行引擎模块**：
   - 在 `backend/app/modules/workflow/` 目录下创建 `engine.py` 文件
   - 实现执行引擎的核心逻辑

2. **实现节点执行器**：
   - 为每种节点类型实现对应的执行器
   - 开始节点：生成初始输入数据
   - AI 节点：调用 AI 模型，处理输入数据，生成输出数据
   - 结束节点：处理最终输出数据

3. **实现工作流执行逻辑**：
   - 解析工作流结构，确定执行顺序
   - 按顺序执行节点
   - 处理节点间的数据传递
   - 更新执行状态和日志

### 2. 集成执行引擎到 API

1. **修改执行 API**：
   - 在 `routes.py` 中修改 `execute_workflow` 函数
   - 调用执行引擎执行工作流
   - 返回执行结果

2. **添加执行状态更新 API**：
   - 实现获取执行状态的 API
   - 实现获取执行日志的 API

### 3. 前端集成

1. **添加执行按钮**：
   - 在工作流详情页添加执行按钮
   - 点击后调用执行 API

2. **添加执行状态和日志展示**：
   - 显示执行状态（pending、running、success、error）
   - 显示执行日志，包括每个节点的输入输出数据

### 4. 测试和优化

1. **创建测试工作流**：
   - 创建包含开始节点、AI 节点和结束节点的工作流
   - 配置 AI 节点的模型和提示词

2. **测试执行流程**：
   - 执行工作流，验证执行引擎是否正常工作
   - 检查执行日志是否正确显示

3. **优化执行引擎**：
   - 处理错误情况
   - 优化执行性能
   - 增强执行引擎的可扩展性

## 技术实现要点

### 执行引擎核心逻辑

- **工作流解析**：使用拓扑排序确定节点执行顺序
- **数据传递**：通过边传递数据，支持节点间的数据流动
- **状态管理**：实时更新执行状态和日志
- **错误处理**：捕获和处理执行过程中的错误

### AI 节点实现

- **模型调用**：使用 OpenAI API 或其他 AI 服务
- **提示词处理**：支持模板化提示词，替换输入变量
- **结果处理**：解析 AI 返回结果，转换为标准格式

### 前端集成

- **执行状态展示**：使用实时更新显示执行状态
- **日志展示**：使用表格或列表展示执行日志
- **用户交互**：提供执行、取消等操作按钮

## 开发时间估计

- 执行引擎核心逻辑：1-2 天
- API 集成：半天
- 前端集成：1 天
- 测试和优化：半天

## 预期结果

- 用户可以创建包含开始节点、AI 节点和结束节点的工作流
- 用户可以点击执行按钮启动工作流
- 执行引擎会按照顺序执行节点
- AI 节点会调用预设的 AI 模型处理数据
- 执行结果会显示在执行日志中
- 工作流执行完成后会显示成功状态
