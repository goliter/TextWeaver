## 推荐布局方案

### **方案1：分层布局 + 侧边分支**（最推荐）

```
┌─────────────────────────────────────┐
│  📄 输入节点                   │
│         ↓                         │
│  🤖 AI节点 (主线)               │
│         ↓                         │
│  📝 输出节点                   │
└─────────────────────────────────────┘
    ↙           ↘
┌──────────┐  ┌──────────┐
│📄文件读取│  │📄文件写入│
│(人物)   │  │(人物)   │
└──────────┘  └──────────┘
```

**实现方式**：
- 主流程节点：居中排列，使用主色调（蓝色/紫色）
- 分支节点：放在左侧或右侧，使用辅助色调（灰色/黄色）
- 通过节点类型或用户标记区分主/分支

### **方案2：dagre + 自定义rank**

```typescript
// 强制主流程节点在中间
dagre.setDefaultNode(node => ({
  rank: isMainNode(node) ? 1 : 2,  // 主流程rank=1，分支rank=2
  ranksep: 100,
}));

// 分支节点按类型分组
const groupedNodes = groupByType(nodes);
```

### **方案3：完全手动 + 智能提示**

- 用户完全手动布局
- 系统提供"对齐到网格"、"水平分布"、"垂直分布"等辅助工具
- 不强制自动布局

## 我的建议

**推荐方案1（分层布局 + 侧边分支）**，因为：

1. **视觉清晰**：主流程一目了然
2. **符合逻辑**：主线和辅助功能分离
3. **灵活可控**：用户可以手动微调
4. **易于扩展**：未来可以添加更多分支类型

**实现细节**：
- 在节点配置中添加"是否主流程"选项
- 主流程节点：x坐标居中（如400px）
- 分支节点：x坐标偏移（如200px或600px）
- y坐标按层级自动计算
