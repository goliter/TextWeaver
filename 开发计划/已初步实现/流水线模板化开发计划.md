# 流水线模板化功能开发计划

## 项目概述

实现工作流模板化功能，允许用户将当前工作流保存为模板，利用模板快速创建新的工作流，并支持模板分享功能。

## 开发阶段

### 阶段1：模板数据模型（预计1-2天）

#### 1.1 数据库模型设计

**文件位置**：`backend/app/modules/template/models.py`

**模型列表**：
- `WorkflowTemplate` - 模板主表
- `TemplateNode` - 模板节点表
- `TemplateEdge` - 模板边（连接）表
- `TemplateFile` - 模板文件表
- `TemplateShare` - 模板分享表

#### 1.2 Schema定义

**文件位置**：`backend/app/modules/template/schemas.py`

**Schema列表**：
- `WorkflowTemplateCreate` - 创建模板请求
- `WorkflowTemplateResponse` - 模板响应
- `WorkflowTemplateList` - 模板列表项
- `TemplateShareCreate` - 创建分享请求
- `TemplateShareResponse` - 分享响应

#### 1.3 Alembic迁移

- 生成迁移脚本
- 执行数据库迁移

## 技术要点

### 数据库关系图

```
WorkflowTemplate
├── TemplateNode (1:N)
├── TemplateEdge (1:N)
├── TemplateFile (1:N)
└── TemplateShare (1:N)
```

### 文件系统复制策略

1. **保存模板时**：
   - 递归遍历工作流的文件系统
   - 将文件夹和文件内容复制到 TemplateFile 表
   - 保持文件层级结构

2. **使用模板时**：
   - 为新工作流创建根文件夹
   - 递归复制 TemplateFile 到新工作流的文件系统
   - 更新节点中的文件路径引用

### 分享机制

1. **分享Token生成**：
   - 使用 UUID 或随机字符串生成唯一分享标识
   - 存储分享信息（模板ID、分享者、权限、过期时间）

2. **权限控制**：
   - `public` - 公开分享，任何人可访问
   - `private` - 私有分享，需要特定链接

3. **分享链接格式**：
   - `/shared/template/{share_token}`

## 注意事项

1. **数据一致性**：确保模板数据与工作流数据的一致性
2. **性能优化**：大量文件复制时考虑异步处理
3. **权限控制**：确保用户只能访问自己的模板
4. **错误处理**：完善的错误处理和用户提示
5. **测试覆盖**：为关键功能编写单元测试
